# Code Dispatcher - 多后端任务执行器

Code Dispatcher 是一个多后端任务分发器，统一调度 `codex`、`claude`、`gemini` 三个 AI 编码工具。

```
┌─────────────────────────────────────────────────────────────────┐
│                     Code Dispatcher 架构                         │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   用户请求 ──→ ┌─────────────┐ ──→ ┌─────────┐ ──→ 结果返回     │
│                │  Backend    │     │ Codex   │                  │
│                │  Router     │     │ Claude  │                  │
│                │             │     │ Gemini  │                  │
│                └─────────────┘     └─────────┘                  │
│                       │                                         │
│           ┌───────────┼───────────┐                             │
│           ▼           ▼           ▼                             │
│   ┌──────────┐  ┌──────────┐  ┌──────────┐                      │
│   │ Task A   │  │ Task B   │  │ Task C   │   ← --parallel       │
│   │ backend  │  │ backend  │  │ backend  │      DAG调度          │
│   │ :codex   │  │ :claude  │  │ :gemini  │                      │
│   └──────────┘  └──────────┘  └──────────┘                      │
│        │             │             │                            │
│        └─────────────┴─────────────┘                            │
│                      │                                          │
│                      ▼                                          │
│              ┌─────────────┐                                    │
│              │  Session ID │  ← --resume 恢复                   │
│              └─────────────┘                                    │
│                                                                 │
│   三种模式: 单任务 / 并行执行 / 断点续传                         │
│   关键规则: 禁止默认终止 code-dispatcher 进程                    │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 三种后端定位

| 后端 | 定位 | 最佳场景 |
|------|------|---------|
| **codex** | 复杂开发 | 复杂逻辑、bug 修复、优化重构、大规模代码生成 |
| **claude** | 快速响应 | 快速修复、代码 review、补充分析、文档编写 |
| **gemini** | UI 原型 | 前端 UI/UX 原型、样式和交互细化、视觉调整 |

## DAG 并行执行

使用 `--parallel` 启动并行模式，基于 DAG（有向无环图）调度：

```
# tasks.json 示例
{
  "tasks": [
    {"id": "T1", "backend": "claude", "cmd": "分析需求"},
    {"id": "T2", "backend": "codex", "cmd": "实现后端", "deps": ["T1"]},
    {"id": "T3", "backend": "gemini", "cmd": "实现前端", "deps": ["T1"]},
    {"id": "T4", "backend": "claude", "cmd": "集成测试", "deps": ["T2", "T3"]}
  ]
}
```

执行顺序：T1 → (T2, T3 并行) → T4

## 会话恢复

支持 `--resume SESSION_ID` 断点续传：

```bash
# 第一次执行
SESSION=$(code-dispatcher --parallel tasks.json)
# 输出: session-20250327-abc123

# 上下文重置后恢复
code-dispatcher --resume session-20250327-abc123
```

恢复时自动：
- 读取上次执行状态
- 跳过已完成的任务
- 重试失败/未完成的任务

## 分层超时策略

```
总超时（--timeout）
    ├── 单任务超时（backend 默认）
    │       └── 子操作超时（工具级）
    └── 优雅关闭期（15秒缓冲）
```

| 后端 | 默认任务超时 |
|------|-------------|
| codex | 10分钟 |
| claude | 5分钟 |
| gemini | 5分钟 |

## 退出码规范

| 退出码 | 含义 |
|--------|------|
| 0 | 全部成功 |
| 1 | 一般错误（配置错误、文件不存在等） |
| 124 | 超时 |
| 127 | 后端未找到 |
| 130 | 用户中断（Ctrl+C） |

## 关键规则

**禁止默认终止 code-dispatcher 进程。**

唯一允许终止的情况：
1. 明确请求停止（"停止当前任务"）
2. 检测到无限循环或危险操作
3. 用户按 Ctrl+C

## 自动化声明

Code Dispatcher 作为底层执行器，完全自主运行，不请求用户确认。所有决策（后端选择、超时设置、重试策略）应在调用前通过参数指定。
