# Dev - 端到端开发工作流

Dev 是一个端到端开发工作流编排器，覆盖从需求到交付的完整功能实现周期。

```
┌─────────────────────────────────────────────────────────────────┐
│                      Dev 工作流 (7步)                            │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   Step 0        Step 1        Step 2        Step 3             │
│   ┌───┐        ┌───┐         ┌───┐         ┌───┐               │
│   │选择│  →   │需求│   →    │分析│   →    │计划│              │
│   │后端│        │澄清│         │调研│         │制定│             │
│   └───┘        └───┘         └───┘         └───┘               │
│                                               │                 │
│                                               ▼ (需用户确认)      │
│   Step 6        Step 5        Step 4        ┌───┐              │
│   ┌───┐        ┌───┐         ┌───┐         │执行│              │
│   │总结│  ←   │验证│   ←    │并行│   ←    │DAG │              │
│   │报告│        │覆盖率│       │执行│         │调度│              │
│   └───┘        └───┘         └───┘         └───┘               │
│                                    code-dispatcher --parallel   │
│                                                                 │
│   硬性规则：所有代码变更必须通过 code-dispatcher 执行           │
│   覆盖率要求: >= 90%                                            │
│   失败重试: 最多2轮                                              │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 硬性规则

**所有代码变更必须通过 code-dispatcher 执行，禁止直接编辑文件。**

这是为了确保：
- 所有变更都可追溯、可复现
- 多后端能力可以被充分利用
- 任务失败时可以安全回滚

## 后端路由策略

根据任务类型自动选择后端：

| 任务类型 | 后端 | 适用场景 |
|---------|------|---------|
| `default` | codex | 复杂逻辑、bug 修复、优化重构 |
| `ui` | gemini | 前端 UI/UX 原型、样式和交互细化 |
| `quick-fix` | claude | 快速修复、review、补充分析 |
| `docs` | claude | 文档编写、注释完善 |

## 开发计划模板

每个功能必须先在 `.specs/{feature_name}/dev-plan.md` 制定计划：

```markdown
## 概览
- 需求一句话描述
- 任务类型：[ui/quick-fix/docs/default]
- 参考文件：@file1 @file2

## 任务清单（2-5个）

### Task 1: [任务名称]
- **ID**: T1
- **类型**: [ui/quick-fix/docs/default]
- **描述**: 具体要做什么
- **文件范围**: 要修改的文件
- **依赖**: 无 / T1
- **测试命令**: `npm test -- --testPathPattern=...`
- **测试重点**: 验证什么

### Task 2: ...
```

## 覆盖率验证

执行完成后自动验证：
- 单元测试覆盖率 >= 90%
- 失败任务自动重试（最多2轮）
- 覆盖率不足时阻止提交

## 自动化声明

此工作流完全自动化执行，无需中间确认。例外情况：
- 开发计划制定完成后需要用户确认
- 覆盖率连续2轮不达标时需要人工介入
