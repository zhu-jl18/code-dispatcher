# Skills 与 Bundles 使用指南（总览）

本文集中说明 `code-dispatcher` 体系中可用的 Skills 与 Bundles。


## Skills 清单

### `code-dispatcher`

提供 code-dispatcher 执行器的详细说明，支持 codex、claude、gemini 三个后端统一调度，并提供 `--parallel` 并行执行与 `--resume` 断点续传能力。该 Skill 是其他多数 Skill 的基础依赖。

```text
┌─────────────────────────────────────────────────────────────────┐
│                     code-dispatcher 架构                         │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   用户请求 ──→ ┌─────────────┐ ──→ ┌─────────┐ ──→ 结果返回     │
│                │  Backend    │     │ Codex   │                  │
│                │  Router     │     │ Claude  │                  │
│                │             │     │ Gemini  │                  │
│                └─────────────┘     └─────────┘                  │
│                       │                                         │
│           ┌───────────┼───────────┐                             │
│           ▼           ▼           ▼                             │
│   ┌──────────┐  ┌──────────┐  ┌──────────┐                      │
│   │ Task A   │  │ Task B   │  │ Task C   │   ← --parallel       │
│   │ backend  │  │ backend  │  │ backend  │      DAG调度          │
│   │ :codex   │  │ :claude  │  │ :gemini  │                      │
│   └──────────┘  └──────────┘  └──────────┘                      │
│        │             │             │                            │
│        └─────────────┴─────────────┘                            │
│                      │                                          │
│                      ▼                                          │
│              ┌─────────────┐                                    │
│              │  Session ID │  ← --resume 恢复                   │
│              └─────────────┘                                    │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### `dev`

端到端开发工作流编排器，覆盖完整的功能实现周期。工作流包括：需求澄清、制定开发计划、智能选择后端引擎、基于 DAG 的并行任务执行，以及代码覆盖率验证。

```text
┌─────────────────────────────────────────────────────────────────┐
│                      dev 工作流 (7步)                            │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   Step 0        Step 1        Step 2        Step 3             │
│   ┌───┐        ┌───┐         ┌───┐         ┌───┐               │
│   │选择│  →   │需求│   →    │分析│   →    │计划│              │
│   │后端│        │澄清│         │调研│         │制定│             │
│   └───┘        └───┘         └───┘         └───┘               │
│                                               │                 │
│                                               ▼ (需用户确认)      │
│   Step 6        Step 5        Step 4        ┌───┐              │
│   ┌───┐        ┌───┐         ┌───┐         │执行│              │
│   │总结│  ←   │验证│   ←    │并行│   ←    │DAG │              │
│   │报告│        │覆盖率│       │执行│         │调度│              │
│   └───┘        └───┘         └───┘         └───┘               │
│                                    code-dispatcher --parallel   │
│                                                                 │
│   覆盖率要求: >= 90%                                            │
│   失败重试: 最多2轮                                              │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### `code-council`

多后端并行代码评审方案，同时启动 2~3 个独立的 AI reviewer 进行交叉复核，最后由 host agent 综合各 reviewer 意见进行终审并输出统一报告。

```text
┌─────────────────────────────────────────────────────────────────┐
│                     code-council 评审流程                        │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   目标代码 ──┬──→ ┌──────────┐                                  │
│   @file     │    │ Reviewer │ ──┐                              │
│   diff      │    │  Codex   │   │                              │
│             │    └──────────┘   │                              │
│             │                   │                              │
│             ├──→ ┌──────────┐   │    ┌──────────────┐          │
│             │    │ Reviewer │   ├──→ │ Host Agent   │          │
│             │    │  Claude  │   │    │              │          │
│             │    └──────────┘   │    │ 1. 去重      │          │
│             │                   │    │ 2. 验证      │          │
│             ├──→ ┌──────────┐   │    │ 3. 置信评分   │          │
│                  │ Reviewer │   │    │ 4. 合成报告   │          │
│                  │  Gemini  │ ──┘    └──────────────┘          │
│                  └──────────┘            │                     │
│                                          ▼                     │
│                              ┌─────────────────────┐           │
│                              │ Code Council 评审报告 │           │
│                              │ - CRITICAL          │           │
│                              │ - WARNING           │           │
│                              │ - INFO              │           │
│                              └─────────────────────┘           │
│                                                                 │
│   置信规则: ≥2个reviewer报告 → 高置信度                          │
│   仅1个报告 → 需host验证代码证据                                 │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### `github-issue-pr-flow`

完整的 Issue 到 PR 闭环交付流程，自动完成需求拆解、代码实现、提交 PR、应对 code review、以及最终的 squash-merge 合并。

```text
┌─────────────────────────────────────────────────────────────────┐
│                    github-issue-pr-flow 流程                     │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   Phase 0      Phase 1       Phase 2       Phase 3              │
│   ┌───┐       ┌───┐        ┌───┐        ┌───┐                 │
│   │同步│  →   │拆解│   →   │实现│   →   │提PR│                │
│   │基线│       │Issue│       │分支│        │关联│                │
│   └───┘       └───┘        └───┘        └───┘                 │
│     │                                          │                │
│     │    Phase 6      Phase 5       Phase 4    │                │
│     │    ┌───┐       ┌───┐        ┌───┐       │                │
│     └──→ │合并│  ←   │处理│   ←   │收集│ ←─────┘                │
│          │关闭│       │Review│       │信号│                      │
│          └───┘       └───┘        └───┘                      │
│            │                                            │      │
│            └────────────────────────────────────────────┘      │
│                          循环 (最多3轮)                          │
│                                                                 │
│   squash-merge ──→ 删除远程分支 ──→ 验证issue关闭                │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### `pr-review-reply`

专门用于处理自动化 bot review（如 Gemini、CodeRabbit 等）的 triage 工作流。自动读取 review 评论、验证每个发现、决定修复或反驳，并在对应的 GitHub review thread 中回复和解决。

```text
┌─────────────────────────────────────────────────────────────────┐
│                    pr-review-reply 处理流程                      │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   Step 1: 获取评论                                               │
│   ┌──────────────────────────────────────────┐                 │
│   │  Source 1: Review body (<details> blocks) │                 │
│   │  Source 2: Line-level comment threads     │                 │
│   │  Source 3: CI status                      │                 │
│   └──────────────────────────────────────────┘                 │
│                      │                                          │
│                      ▼                                          │
│   Step 2: 验证代码 ──→ 读取文件 ──→ 检查CI ──→ 交叉验证           │
│                      │                                          │
│                      ▼                                          │
│   Step 3: 决策      ┌─────────┐    ┌─────────┐                 │
│                     │  Fix    │ or │ Rebut   │                 │
│                     │ 修复     │    │ 反驳    │                 │
│                     └────┬────┘    └────┬────┘                 │
│                          │              │                      │
│                          ▼              ▼                      │
│   Step 4: 本地修复    修改代码      收集证据                    │
│           (不push)     │           草拟回复                     │
│                        │                                      │
│                        └──────────┬───────────────────────────┘
│                                   │
│                                   ▼
│   Step 5: 回复线程 ←── gh api 回复评论                          │
│                                   │
│                                   ▼
│   Step 6: 解决线程 ←── GraphQL resolveReviewThread               │
│                                   │
│           所有处理完毕 ──→ 批量push ──→ 等待re-review            │
│                                                                 │
│   循环限制: 最多3轮                                               │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```
