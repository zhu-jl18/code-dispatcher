# Code Council - 多视角并行代码评审

Code Council 是一个多后端并行代码评审方案，同时启动 2~3 个独立的 AI reviewer 进行交叉复核，最后由 host agent 综合各 reviewer 意见进行终审。

```
┌─────────────────────────────────────────────────────────────────┐
│                     Code Council 评审流程                        │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   目标代码 ──┬──→ ┌──────────┐                                  │
│   @file     │    │ Reviewer │ ──┐                              │
│   diff      │    │  Codex   │   │                              │
│             │    └──────────┘   │                              │
│             │                   │                              │
│             ├──→ ┌──────────┐   │    ┌──────────────┐          │
│             │    │ Reviewer │   ├──→ │ Host Agent   │          │
│             │    │  Claude  │   │    │              │          │
│             │    └──────────┘   │    │ 1. 去重      │          │
│             │                   │    │ 2. 验证      │          │
│             │                   │    │ 3. 置信评分   │          │
│             │                   │    │ 4. 合成报告   │          │
│             ├──→ ┌──────────┐   │    └──────────────┘          │
```

## 为什么需要多视角评审

单一 reviewer 存在盲区：
- **Codex** 擅长逻辑漏洞，但可能忽视安全细节
- **Claude** 擅长代码质量，但可能遗漏边界情况
- **Gemini** 擅长架构问题，但可能误判复杂度

2-3 人并行评审可实现交叉验证，提高问题发现率。

## 6大审查维度

每个 reviewer 必须检查以下维度：

| 维度 | 检查重点 | 常见错误 |
|------|---------|---------|
| **安全漏洞** | SQL注入、XSS、敏感信息泄露 | 未转义的用户输入 |
| **逻辑错误** | 条件判断、循环边界、状态转换 | 边界条件遗漏 |
| **性能问题** | 算法复杂度、资源泄漏、热点代码 | O(n²) 循环嵌套 |
| **错误处理** | 异常捕获、错误传播、降级策略 | 空指针未处理 |
| **代码质量** | 可读性、命名规范、重复代码 | 魔法数字、过长函数 |
| **边界情况** | 空值、极限值、并发场景 | 未考虑空数组 |

## 置信度评分规则

```
≥2 个 reviewer 报告同一问题  →  高置信度（直接采纳）
仅 1 个 reviewer 报告        →  需 host agent 验证代码证据
无人报告但 host 发现         →  补充报告
```

## 严重级别定义

| 级别 | 定义 | 处理要求 |
|------|------|---------|
| **CRITICAL** | 会导致生产故障、数据丢失、安全漏洞 | 必须修复，阻塞合并 |
| **WARNING** | 代码质量问题、潜在风险 | 建议修复，可讨论 |
| **INFO** | 风格建议、优化提示 | 仅供参考 |

## Host Agent 综合流程

1. **去重**：合并 reviewer 的重复发现
2. **验证**：检查低置信度问题的代码证据
3. **评分**：按 CRITICAL/WARNING/INFO 分类
4. **合成**：生成统一的评审报告

## 报告格式

```markdown
## Code Council 评审报告

### CRITICAL (必须修复)
- [ ] **[安全]** SQL注入风险 @file:42 - 使用参数化查询

### WARNING (建议修复)
- [ ] **[性能]** O(n²) 循环 @file:88 - 考虑使用哈希表优化

### INFO (参考)
- 函数名可更清晰 @file:15
```

## 自动化声明

Code Council 作为代码审查工具，完全自动化执行，不请求用户确认。评审报告直接输出，由开发者决定是否采纳建议。
